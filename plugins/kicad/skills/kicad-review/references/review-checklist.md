# KiCad 범용 회로도 리뷰 체크리스트

> 이 체크리스트는 도메인에 무관하게 모든 KiCad 회로도에 적용 가능한 검증 항목이다.
> 도메인별 추가 항목은 SKILL.md의 "도메인별 추가 검증" 섹션을 참조한다.

## 판정 기준

| 등급 | 의미 | 조치 |
|------|------|------|
| ERROR | 하드웨어 손상 또는 동작 불가 | 반드시 수정 후 재검증 |
| WARN | 동작하지만 잠재적 문제 | 문서화 또는 개선 권장 |
| OK | 이상 없음 | - |
| N/A | 해당 회로에 미적용 | - |

---

## A. 전원 설계 (8항목)

| ID | 항목 | ERROR 조건 | 검증 방법 |
|----|------|-----------|----------|
| A1 | 모든 IC의 VCC/VDD 핀이 전원 네트에 연결되어 있는가 | 전원 핀 미연결 (플로팅) | 각 IC의 전원 핀을 네트리스트에서 추적 |
| A2 | 모든 IC의 GND/VSS 핀이 GND 네트에 연결되어 있는가 | GND 핀 미연결 | 같은 방법 |
| A3 | 각 IC의 VCC 핀 근처에 디커플링 캡(100nF)이 있는가 | 디커플링 캡 누락 | IC 주변 캡 탐색 |
| A4 | 벌크 캡(10μF~47μF)이 전원 입력부에 있는가 | WARN: 전원 리플 | 전원 커넥터/레귤레이터 출력 근처 |
| A5 | 전원 전압이 각 IC의 허용 범위 이내인가 | 정격 초과 → IC 손상 | 데이터시트 Abs Max 대조 |
| A6 | 전원 시퀀싱이 필요한 IC의 EN 핀이 올바르게 연결되었는가 | 시퀀싱 오류 → 래치업 | 데이터시트 Power-up Sequence 참조 |
| A7 | PWR_FLAG가 전원 네트에 배치되어 있는가 | WARN: KiCad ERC 경고 | 전원 네트에 PWR_FLAG 심볼 존재 확인 |
| A8 | 서로 다른 전압 레일이 직접 단락되지 않았는가 | 전원 단락 → 보드 손상 | 네트 이름으로 전압 레일 분리 확인 |

## B. 핀 연결 완전성 (6항목)

| ID | 항목 | ERROR 조건 | 검증 방법 |
|----|------|-----------|----------|
| B1 | IC의 필수 핀(VCC, GND, 기능 핀)이 모두 연결되었는가 | 필수 핀 미연결 | 데이터시트 핀 설명에서 "Must be connected" 확인 |
| B2 | NC(No Connect) 표시 없이 미연결된 핀이 있는가 | WARN: 의도적 NC인지 불명확 | KiCad no_connect 플래그 확인 |
| B3 | 입력 핀이 플로팅 상태가 아닌가 (풀업/풀다운 또는 구동원 필요) | 플로팅 입력 → 불안정 동작 | 입력 핀의 네트에 구동원 존재 확인 |
| B4 | 오픈 드레인/오픈 콜렉터 출력에 풀업 저항이 있는가 | 풀업 누락 → 통신 불가 | I2C SDA/SCL, INT# 핀 등 |
| B5 | 미사용 입력 핀이 적절히 처리되었는가 (GND 또는 풀다운) | WARN: 플로팅 | 미사용 핀 목록 대조 |
| B6 | 양방향 버스의 방향 제어 핀이 연결되었는가 (DIR, OE) | 버스 충돌 가능 | 레벨 시프터, 버퍼 IC |

---

## C. 신호 무결성 (6항목)

| ID | 항목 | ERROR 조건 | 검증 방법 |
|----|------|-----------|----------|
| C1 | 고속 신호(>50MHz)에 직렬 종단 저항이 있는가 | WARN: 링잉/반사 | 클럭, 데이터 버스 라인 |
| C2 | 차동 페어(USB, Ethernet, PCIe)가 쌍으로 연결되었는가 | 차동 쌍 불완전 | P/N 쌍의 네트 추적 |
| C3 | 차동 신호에 AC 커플링 캡이 필요한 경우 배치되었는가 | WARN: DC 바이어스 문제 | PCIe TX/RX, GT 트랜시버 |
| C4 | 클럭 신호가 여러 부하에 팬아웃 시 버퍼가 있는가 | WARN: 신호 품질 | 클럭 분배 네트 |
| C5 | 아날로그 신호 경로에 디지털 노이즈 차단(필터)이 있는가 | WARN: ADC 정밀도 저하 | ADC 입력, 기준 전압 |
| C6 | ESD 보호가 외부 커넥터 핀에 있는가 | WARN: ESD 취약 | USB, Ethernet, GPIO 헤더 |

## D. 데이터시트 정합성 (8항목)

| ID | 항목 | ERROR 조건 | 검증 방법 |
|----|------|-----------|----------|
| D1 | IC 파트넘버가 회로도 심볼에 정확히 명시되어 있는가 | 파트넘버 불명/불일치 | Value 필드 확인 |
| D2 | 풋프린트가 파트넘버의 패키지와 일치하는가 | 패키지 불일치 | Footprint 필드 vs 데이터시트 Package |
| D3 | 데이터시트 Application Circuit의 필수 외부 부품이 모두 있는가 | 필수 부품 누락 | 데이터시트 Typical Application |
| D4 | 외부 부품 값(저항, 캡)이 데이터시트 권장값과 일치하는가 | WARN: 부품값 불일치 | 데이터시트 권장 회로의 부품값 |
| D5 | 정격 전압/전류를 초과하지 않는가 | 정격 초과 → 손상 | Absolute Maximum Ratings |
| D6 | 동작 온도 범위가 사용 환경과 일치하는가 | WARN: 온도 범위 부적합 | Operating Conditions |
| D7 | 핀 기능(입력/출력/양방향)이 회로도 연결과 일치하는가 | 핀 기능 오용 | Pin Description 테이블 |
| D8 | 스트랩/설정 핀이 원하는 모드로 올바르게 구성되었는가 | 모드 설정 오류 | 부트 모드, 인터페이스 선택 핀 |

## E. ERC 수준 검증 (5항목)

| ID | 항목 | ERROR 조건 | 검증 방법 |
|----|------|-----------|----------|
| E1 | 동일 네트에 2개 이상의 출력이 연결되지 않았는가 | 출력 충돌 → 단락 | 네트별 핀 타입 분석 |
| E2 | 전원 출력(power_out)이 전원 입력(power_in)에 연결되었는가 | WARN: ERC 경고 | 전원 네트 핀 타입 |
| E3 | 네트 이름이 일관적인가 (대소문자, 언더스코어 통일) | WARN: 네트 분리 위험 | 네트 이름 패턴 |
| E4 | 글로벌 레이블과 로컬 레이블이 혼용되지 않는가 | WARN: 서브시트 간 연결 오류 | 시트 간 레이블 타입 |
| E5 | 모든 와이어가 정확히 교차점(junction)에서 연결되는가 | WARN: 시각적 연결이 실제 미연결 | KiCad junction 마커 |

## L. 제조사 공식 라이브러리 비교 (5항목)

제조사가 제공하는 공식 KiCad 라이브러리(SnapEDA, Ultra Librarian 등)와 회로도 심볼을 대조하여, 커스텀 심볼의 핀 오류를 사전에 차단한다.

| ID | 항목 | ERROR 조건 | 검증 방법 |
|----|------|-----------|----------|
| L1 | 핀 번호가 공식 라이브러리와 일치하는가 | 핀 번호 불일치 → 잘못된 패드에 연결 | 공식 심볼의 pin number vs 회로도 심볼 |
| L2 | 핀 이름이 공식 라이브러리와 일치하는가 | WARN: 이름 차이 (기능 동일하면) | 공식 심볼의 pin name vs 회로도 심볼 |
| L3 | 핀 총 개수가 일치하는가 | 핀 수 불일치 → 심볼 오류 | 공식 심볼의 총 핀 수 vs 회로도 심볼 |
| L4 | 풋프린트가 파트넘버의 패키지와 일치하는가 | 패키지 불일치 → 실장 불가 | 공식 풋프린트 vs 회로도 Footprint 필드 |
| L5 | 핀 타입(입력/출력/전원/양방향)이 올바르게 정의되었는가 | 전원 핀이 I/O로 정의됨 → ERC 미검출 | 공식 심볼의 pin type vs 회로도 심볼 |

**공식 라이브러리 검색 순서**: KiCad 내장 → SnapEDA → Ultra Librarian → Component Search Engine → 제조사 공식

## V. PDF 시각적 검토 (7항목)

kicad-cli로 PDF 내보내기 후 멀티모달 분석으로 검증한다. 텍스트 파싱으로는 감지하기 어려운 시각적 오류를 잡는 단계이다.

| ID | 항목 | ERROR 조건 | 검증 방법 |
|----|------|-----------|----------|
| V1 | 교차하는 와이어에 junction(접합점) 마커가 있는가 | 마커 없으면 전기적 미연결 | PDF에서 와이어 교차점 시각 확인 |
| V2 | 부품 심볼, 레이블, 값 텍스트가 서로 겹치지 않는가 | WARN: 가독성 저하 | PDF에서 텍스트/심볼 겹침 확인 |
| V3 | 네트 레이블이 와이어 끝에 정확히 연결되어 있는가 | 미세 이탈 시 미연결 | PDF에서 레이블-와이어 접점 확인 |
| V4 | 전원 심볼(VCC/GND)이 와이어에 올바르게 연결되어 있는가 | 떠 있는 전원 심볼 | PDF에서 전원 심볼 연결 확인 |
| V5 | 신호 흐름이 입력→왼쪽, 출력→오른쪽 관례를 따르는가 | WARN: 가독성 | PDF에서 전체 흐름 방향 확인 |
| V6 | 시트 간 계층 핀과 글로벌 레이블이 시각적으로 일관적인가 | WARN: 혼동 가능 | 루트 시트의 계층 핀 vs 서브시트 레이블 |
| V7 | 끊어진 와이어, 허공에서 끝나는 미완성 연결이 없는가 | 미배선 → 회로 미완성 | PDF에서 와이어 끝점 확인 |

## 검증 실행 순서

효율적인 검증을 위한 권장 순서:

1. **V (시각적 검토)** → PDF로 전체 구조와 시각적 오류를 먼저 파악
2. **L (라이브러리 비교)** → 심볼 핀 오류를 조기에 차단
3. **A (전원)** → 전원이 없으면 나머지 의미 없음
4. **D (데이터시트)** → IC별 권장 회로 파악이 다른 검증의 기준
5. **B (핀 연결)** → 데이터시트 기준으로 필수 핀 파악 후 검증
6. **C (신호 무결성)** → 고속 신호 관련
7. **E (ERC)** → 최종 정합성 확인
8. **F (도메인별)** → 해당 시에만 추가 적용

---

## 보고서 작성 원칙

- ERROR 항목에는 반드시 **데이터시트 근거**(섹션명, 테이블 번호, 페이지)를 명시한다
- "권장" 또는 "~인 것 같다" 같은 모호한 표현 대신, 데이터시트 원문을 인용한다
- 수정 조치는 구체적으로 제시한다 (예: "C5를 100nF에서 1μF로 변경" ← 이렇게)
- 확인할 수 없는 항목은 N/A로 표시하되 사유를 기술한다
